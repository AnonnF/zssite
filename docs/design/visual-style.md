# ZSsite 视觉风格指南

> 本文档是 ZSsite 的视觉设计单一事实来源（single source of truth）。实现 UI 前请先阅读本文档；Cursor 规则 `.cursor/rules/zssite-design.mdc` 会引用本文件。

---

## 1. 项目气质

ZSsite 是 **施展 / Zhan Shi** 的个人网站，同时承担 **工程项目档案** 的角色。

整体气质应接近：

| 关键词 | 含义 |
|--------|------|
| 个人网站 | 有明确主体，不是产品营销页 |
| 工程档案馆 | 项目可被检索、分类、展开阅读 |
| 信息面板 | 模块化区块、标签、元数据、细线框 |
| 工业海报感 | 粗黑字体、大标题、留白与密度对比 |
| 复古未来感 | 克制、结构化，而非霓虹赛博 |

**不是：**

- 普通 portfolio 模板（三栏技能条 + 圆形头像 + 渐变 CTA）
- SaaS landing page（Hero + 定价 + 客户 logo 墙）
- 游戏官网（宣传视频主导、角色立绘、活动 banner）

**参考 mood（仅布局语言，非品牌复制）：** ARC Raiders 官网的工业海报感、粗黑字体、细线分隔、信息面板布局。详见文末「边界说明」。

## 2. 色彩系统

### 2.1 主色

| 角色 | 色值建议 | 用途 |
|------|----------|------|
| 背景米白 | `#F5F0E8` ~ `#FAF7F2` | 页面主背景、大面积留白 |
| 面板米白 | `#EFEBE3` ~ `#F0ECE4` | 卡片 / 面板底色（与背景有轻微对比即可） |
| 文字黑 | `#1A1A1A` ~ `#111111` | 正文、标题 |
| 分隔线黑 | `#1A1A1A` @ 100% 或 `#000000` @ 12–20% opacity | 1px 细线、边框 |
| 弱化文字 | `#4A4A4A` ~ `#666666` | 元数据、次要说明、时间戳 |

### 2.2  accent（极少使用）

- 默认 **不使用** 彩色 accent。
- 若必须强调（如链接 hover、当前导航项），仅用 **纯黑加深** 或 **下划线 / 边框**，不用蓝紫渐变或高饱和色块。
- 装饰条纹：黑白或米白上的细斜线 / 平行线，透明度低，面积 ≤ 区块 15%。

### 2.3 禁止色

- 蓝紫渐变背景
- 霓虹青 / 品红 / 电蓝
- 大面积彩色 icon 墙
- 玻璃拟态所需的半透明白 + 模糊叠层

## 3. 字体系统

### 3.1 原则

- **中文为主可读内容**；英文可用于小标签、元数据、章节编号（如 `PROJECT ARCHIVE`、`01 / ABOUT`）。
- 视觉冲击力来自 **字重、字号阶梯、行距、分隔线**，而非装饰图形。
- 标题偏 **粗、几何、工业感**；正文偏 **清晰、略紧凑**。

### 3.2 推荐字体栈（实现时可调整，但需保持气质）

| 层级 | 中文 | 英文 / 数字 | 特征 |
|------|------|-------------|------|
| Display / Hero | 思源黑体 Heavy / 阿里巴巴普惠体 Heavy | Druk Wide / Bebas Neue / Oswald（需授权则换开源替代） | 全大写英文标签可选 |
| 标题 H1–H3 | 思源黑体 Bold | Inter / IBM Plex Sans Bold | 字距略紧 |
| 正文 | 思源黑体 Regular / 苹方 / 系统 sans | Inter / IBM Plex Sans | 行高 1.5–1.65 |
| 元数据 / 标签 | 同上，字号 12–13px | 等宽可选：IBM Plex Mono / JetBrains Mono | 全大写 + letter-spacing |

### 3.3 字号阶梯（桌面基准，移动端同比缩小）

| Token | 大小 | 用途 |
|-------|------|------|
| `text-display` | 48–72px | Hero 主标题 |
| `text-h1` | 36–48px | 页面标题 |
| `text-h2` | 24–32px | 区块标题 |
| `text-h3` | 18–20px | 卡片标题 |
| `text-body` | 15–16px | 正文 |
| `text-meta` | 12–13px | 标签、日期、技术栈 |

### 3.4 字重

- 标题：**700–900**
- 正文：**400–500**
- 标签 / 导航：**500–600**，可全大写

## 4. 布局语言

### 4.1 网格与留白

- 内容区最大宽度：**1120–1280px**，居中。
- 水平 padding：**24px（mobile）→ 48–64px（desktop）**。
- 区块垂直间距：**64–96px**（大节之间）；节内组件间距 **24–32px**。
- 优先 **左对齐编辑型排版**，避免居中堆砌式 portfolio 布局。

### 4.2 结构模式

1. **全宽节 + 内部网格**：Hero、项目列表可用 12 栏网格。
2. **信息面板**：矩形区域 + 1px 边框 + 内 padding，可并排 2–3 列。
3. **细线分隔**：用 `border-top` / `border-left` 切分内容，而非卡片阴影。
4. **标签轨（label rail）**：区块左上角或左侧竖排小字英文标签（如 `PROJECTS`、`CAPABILITY`）。

### 4.3 密度

- 首页 Hero 可 **疏**；项目档案区可 **密**（多行元数据、tech tag）。
- 密度通过 **字号对比 + 分隔线** 控制，不用彩色背景区分。

### 4.4 响应式

- Mobile：单列堆叠；面板边框保留；Hero 字号降至 32–40px。
- 避免 mobile 上圆角大卡片 + 浮动 CTA 按钮（典型 SaaS 移动端）。

## 5. 卡片 / 面板样式

### 5.1 标准面板（Project Panel）

```
┌─ 1px solid #1A1A1A ─────────────────────┐
│ [META] 2024 · WEB · TS                  │
│ ─────────────────────────────────────── │  ← 1px 内部分隔
│ 项目标题（Bold）                          │
│ 简短描述正文…                             │
│ [tag] [tag] [tag]                       │
└─────────────────────────────────────────┘
```

- **圆角**：0–4px（默认 **0 或 2px**）
- **阴影**：默认无；hover 可用 `border-color` 加深或 1px 位移，不用 `box-shadow: 0 8px 32px`
- **背景**：与页面背景同系米白，或浅一档
- **边框**：1px solid black / near-black
- **内边距**：20–24px

### 5.2 标签（Tag / Chip）

- 描边样式：`1px solid`，透明或米白底，文字黑色
- 不用 pill 大圆角（`rounded-full` + 彩色底）

### 5.3 按钮

- Primary：黑底白字 **或** 黑边框透明底；hover 反色
- 不用渐变、不用 glow
- 高度 40–44px，直角或 2px 圆角

### 5.4 装饰条纹（可选）

- 仅用于 Hero 背景或节与节之间的 **过渡带**
- 低对比黑白条纹；不可抢正文

## 6. 关键区块设计方向

### 6.1 Header

- 高度：**56–64px**，底部分隔线 1px
- 左：站点标识 `ZSsite` 或 `施展`（字标，非复杂 logo）
- 中/右：导航链接（首页 / 项目 / 关于 / 联系），字重 medium，无 icon 堆砌
- 可选右侧：`EN` 语言切换占位（第一版可隐藏）
- 固定或 sticky 均可；背景米白实色，不用 blur header

### 6.2 Hero（Home）

- 结构：**大标题 + 一行副说明 + 可选元数据行**
- 示例层次：
  - 小标签：`PERSONAL SITE / ENGINEERING ARCHIVE`
  - 主标题：`施展` 或 slogan（中文）
  - 副文：1–2 句能力概述
  - 可选：横向细线 + 能力关键词（AI / Web / Systems…）
- 不用：全屏视频背景、3D 模型、粒子效果、巨大渐变按钮

### 6.3 Project Card（列表项）

- 必含字段：**标题、年份/状态、摘要、技术标签**
- 可选：**缩略图**（矩形、无大圆角、可黑白处理）
- 布局：列表式（行 + 分隔线）或网格面板；推荐 **列表式** 以增强档案感
- Hover：下划线 / 边框加深 / 箭头 `→`，不用 scale + shadow

### 6.4 Project Detail

- 顶部：面包屑或返回链接 + 项目编号
- 主体：标题、元数据面板（角色 / 时间 / 栈 / 链接）
- 内容区：Markdown 友好排版（标题、代码块、图片）
- 侧栏（desktop）：目录或相关项目，用竖线分隔

### 6.5 Footer

- 极简：版权、邮箱 / GitHub 链接、一行 sitemap
- 顶部分隔线；不用多列 widget 墙
- 可重复站点元数据标签风格：`© 2026 ZSsite`

## 7. 禁止事项（Do Not）

| 类别 | 禁止 |
|------|------|
| 风格 | 蓝紫渐变、霓虹赛博、玻璃拟态、过度阴影 |
| 组件 | shadcn 默认 aesthetic 照搬（大圆角 Card、muted 灰底、ring focus 风格） |
| 布局 | 居中对称 portfolio 三件套、pricing table、testimonial carousel |
| 资产 | ARC Raiders logo、角色、官方素材、官方文案 |
| 交互 | 无意义 parallax、自动播放 promo 视频、弹窗订阅 |
| 内容 | 假客户 logo 墙、假数据 testimonial |

## 8. ARC Raiders-inspired 边界说明

### 可以借鉴（布局与气质）

- 粗黑大标题与小型全大写英文标签的组合
- 细黑线网格化信息面板
- 工业 / 军事档案式的 **metadata label**（如 `CLASS`、`TYPE`、`REF` 等抽象标签，不用游戏内术语）
- 米白 / Off-white 背景 + 高对比黑色文字
- 海报式排版：一个大视觉锚点 + 周边信息块
- 克制的复古未来 **结构感**（线条、框、编号）

### 不可复制（品牌与资产）

- ARC Raiders / Embark / Nexon 的 logo、icon、角色、武器、场景图
- 官方 slogan、活动 copy、版本号样式上的品牌识别
- 官方 UI 截图像素级复刻
- 游戏特有名词作为本站主内容（除非是你自己的项目名）

### 自检问题

1. 去掉所有文字后，页面是否仍像 ARC 官网截图？→ 若是，则过于接近。
2. 是否用了黑 + 米白 + 细线 + 面板就能成立？→ 应是。
3. 把「施展 / 工程项目」代入后是否自然？→ 应是。

## 9. 与代码的衔接（预留）

实现阶段建议 tokens 命名：

- `--color-bg`, `--color-surface`, `--color-text`, `--color-border`, `--color-muted`
- `--font-display`, `--font-body`, `--font-mono`
- `--space-section`, `--space-panel`, `--radius-sm`（0–4px）

内容层使用数据驱动（项目 JSON / MDX），样式层遵守本指南。
